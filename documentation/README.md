## Contributing

When adding new features or making significant architectural decisions:

1. **Document the approach** - Create a lessons document explaining:
   - Problem statement
   - Solution approach
   - Implementation patterns
   - Common pitfalls
   - File references with line numbers

2. **Use consistent structure** - Follow the template from existing documentation

3. **Update this README** - Add links to new documentation
